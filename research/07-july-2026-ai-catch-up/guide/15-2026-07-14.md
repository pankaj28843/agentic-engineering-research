# 2026-07-14, Tuesday: Agent trust boundaries fail

<div class="daily-brief" markdown="1">

**Daily brief:** High signal. Direct traces and an incident update exposed Grok
Build's default upload boundary, while a separate Cursor disclosure showed that
opening a Windows repository could execute attacker-controlled code.

<article class="lead-story" markdown="1">

## Grok Build wire traces expose code and history uploads

A wire-level analysis observed Grok Build sending files it read, including `.env` content, through response payloads and session archives. It also observed repository storage that included unread tracked files and Git history; in a 12 GB test repository, at least 5.10 GiB moved before the server returned HTTP 200. A July 14 update said xAI disabled the upload path server-side, clarified that `/privacy` affected retention rather than blocking transmission, and recorded a commitment to delete earlier uploads. A separate user reported a whole-home upload, but that broader scope remains an attributable report rather than independently reproduced evidence.

<div class="impact" markdown="1">

**Builder impact:** Run coding agents with least-privilege filesystem mounts, explicit egress controls, secret isolation, and wire-level tests that distinguish transmission from retention settings.

</div>

<div class="caveat" markdown="1">

**Caveat:** The researcher directly observed transfers; the server-side response and deletion commitment are reported in that researcher's update, not a separately captured xAI notice.

</div>

<div class="discussion" markdown="1">

**Discussion:** Practitioners supplied concrete `bubblewrap`, root-filesystem, and selective-socket patterns for containing local agents.

</div>

## What else mattered

<article markdown="1">

### Cursor repeatedly executes a repository-root `git.exe` on Windows

Mindgard disclosed that opening a repository in Cursor on Windows caused the IDE to discover and repeatedly run a workspace-root `git.exe` without a prompt. Its harmless proof renamed Calculator, and a Process Monitor trace recorded `Cursor.exe` creating the process. The report says Mindgard first disclosed the issue on December 15, HackerOne reproduced it in January, and the latest tested version remained affected after more than six months and 197 releases. The July 14 material delta is public full disclosure, not initial discovery.

<div class="impact" markdown="1">

**Builder impact:** Open untrusted Windows repositories only in a disposable environment; managed fleets should deny executable launches from workspace paths until a verified fix exists.

</div>
