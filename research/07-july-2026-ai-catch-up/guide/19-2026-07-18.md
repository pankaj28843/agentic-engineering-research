## Codex metadata changes its usable-context allocation

OpenAI merged refreshed Codex model metadata into release 0.144, establishing a
real primary-repository change. The rendered diff did not expose numeric values;
a later HN submission identified a reduction from 372,000 to 272,000 usable
tokens, consistent with a 400,000-token window reserving 128,000 for output.
Practitioners reported lossy compaction, shared a workaround, and advocated
durable plan files; others noted the performance and cost risks of long context.

<div class="impact" markdown="1">

**Builder impact:** Treat model metadata and context reserves as versioned
runtime dependencies; test compaction, persist plans externally, and budget
output headroom explicitly.

### HalluSquatting turns invented names into a supply-chain path

A secondary report describes HalluSquatting: an agent invents a repository,
package, skill, or MCP identifier, then retrieves attacker-controlled
promptware. Systems should resolve a canonical identity before fetching, treat
retrieval as privileged, sandbox the result, and log an auditable receipt.

<div class="impact" markdown="1">

**Builder impact:** Separate model-generated names from trusted identities, and
require allowlisted resolution plus sandboxing before agents install or execute
retrieved resources.

</div>

**Caveat:** The report links the primary paper and project, but neither was
separately extracted; quantitative claims remain researcher-attributed.
