<div class="daily-brief" markdown="1">

**Daily brief:** High signal. Reproducible binary inspection showed Claude

<article class="lead-story" markdown="1">

## Claude Code's embedded Bun runtime moves to Rust

Simon Willison inspected his installed Claude Code binary and found an embedded
Bun 1.4.0 plus 563 `.rs` source paths, corroborating Bun creator Jarred Sumner's
statement that Claude Code 2.1.181 and later use the Rust port. Bun 1.4.0 was
then a canary rather than a normal tagged release, yet it was already deployed
inside Claude Code. HN discussion examined memory lifecycle, deterministic
compiler guardrails, and Zig-versus-Rust robustness. Commands against the
production artifact make the infrastructure transition directly inspectable.

<div class="impact" markdown="1">

**Builder impact:** Binary-level dependency inspection belongs in release
forensics because embedded runtime changes can alter memory, performance, and
failure behavior without visible product changes.

### GhostWriter report warns of delayed memory poisoning

Tech Xplore reports New Mexico State University research on GhostWriter, a
two-phase attack that stores hidden instructions or false facts in an agent's
long-term memory and activates them during a later legitimate request. The
article attributes roughly 98% injection and 60% activation rates to the tested
agents and describes AM-Sentry screens at memory save and retrieval time.

<div class="impact" markdown="1">

**Builder impact:** Persistent memory needs provenance, write-time validation,
retrieval-time policy checks, and revocation rather than treatment as trusted
conversation history.

</div>
