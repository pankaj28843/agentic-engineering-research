### Cloudflare separates AI crawlers by purpose


Cloudflare added controls that distinguish Search, Agent, and Training use,
announced a September 15 default for new domains, introduced BotBase, and added
a content-use signal. The actual July 1 delta is finer policy enforcement; the
company's earlier one-click blocking and pay-per-crawl launches are context.

<div class="impact" markdown="1">

**Builder impact:** Publishers and agent operators should expect access policy
to vary by declared purpose and should test crawler identity, fallback behavior,
and content-use signals before September defaults arrive.

</div>

### Kimi K2.7 Code enters GitHub Copilot


GitHub made Kimi K2.7 Code generally available in Copilot, describing it as the
service's first selectable open-weight model, hosted through Azure and positioned
as a lower-cost coding option. The release was published July 1; July 2 HN
discussion does not move it to a later edition.

<div class="impact" markdown="1">

**Builder impact:** Teams can compare an open-weight model through a mainstream
managed channel, but should benchmark repository-specific quality, determinism,
context handling, and provider dependencies before changing defaults.

</div>

<div class="caveat" markdown="1">

**Caveat:** Availability and hosting are inspectable distributor facts;
