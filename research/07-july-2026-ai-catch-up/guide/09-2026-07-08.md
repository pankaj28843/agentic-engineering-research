## OpenAI finds roughly 30% of SWE-Bench Pro public is broken

OpenAI audited all 731 tasks after the public split's reported solve rate rose from 23.3% to 80.3% in eight months. Its agent pipeline flagged 200 tasks, while five experienced engineers reviewed each task and judged 249, or 34.1%, broken; OpenAI estimated the true rate at roughly 30%. Problems included underspecified prompts, hidden requirements, flawed tests, and repository state that made valid solutions fail. OpenAI consequently retracted its earlier recommendation to use this public split for frontier coding models. The result is a methodological correction, not evidence that every private or hidden coding evaluation is invalid.

<div class="impact" markdown="1">

**Builder impact:** Treat public benchmark gains as hypotheses until task quality, contamination, verifier behavior, and production-relevant constraints have been audited.

</div>

<div class="discussion" markdown="1">

**Discussion:** Practitioners identified verifier design, hidden requirements, resource-limit manipulation, and the gap between agent-assisted triage and slower human review as distinct evaluation risks.

</div>

### Databricks tests coding agents on its own multi-million-line codebase

Databricks built reviewed tasks from a private codebase spanning more than ten languages, then compared model and harness combinations on capability and cost per completed task. GLM 5.2 with Pi was statistically tied with Opus 4.8 in its setup while costing $1.28 rather than $1.94 per task. The stronger lesson was methodological: harness choice and per-task cost changed the deployment decision even when pass rates looked similar.

<div class="impact" markdown="1">

**Builder impact:** Maintain a private, reviewed task suite and price successful outcomes; public rankings cannot expose your repository, harness, or cost structure.

</div>

<div class="caveat" markdown="1">

**Caveat:** These are Databricks' internal tasks and rollout observations, not a universal model ranking.

</div>

### GPT-Live combines full-duplex conversation with background work

OpenAI launched GPT-Live-1 and a mini variant globally in its product. The system can listen and speak simultaneously, take interruptions, and hand web or more complex work to GPT-5.5 while the conversation continues. An attributable preview user reported an hour-long session and observed that background delegation worked, but the release said developer API access was still coming soon.

<div class="impact" markdown="1">

**Builder impact:** Voice-agent designs can separate a low-latency conversational loop from slower delegated work, but production API integration was not yet available.

</div>

<div class="caveat" markdown="1">

**Caveat:** Product behavior was shipped; API availability and generalized reliability remained prospective vendor claims.
