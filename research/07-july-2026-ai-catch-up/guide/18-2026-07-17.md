## Claude Code question timeout weakens human-in-the-loop control

Olaf Alders documented that Claude Code 2.1.198's `AskUserQuestion` timed out
after 60 seconds and continued without an answer or with partial answers.
Permission prompts did not auto-resolve, but in allowlisted or
bypass-permissions runs the question could be the only remaining human gate.
The behavior was fixed within days. The report is narrow but operationally
important: a prompt presented as a request for information can become an
implicit authorization mechanism, and an automatic product update can change
that behavior without a team revising its harness. The source supplies a
version, reproduction details, risk mechanics, and the subsequent fix.

<div class="impact" markdown="1">

**Builder impact:** Model every human question as a typed gate with explicit
timeout semantics, fail-closed behavior, version pinning, and regression tests
for unattended runs.

</div>

<div class="caveat" markdown="1">

**Caveat:** This is one product-version failure, not evidence that every agent
question or permission system behaves the same way.
